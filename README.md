# Lovofeed

研究室のマスコットを育てる Web アプリです。
研究室の在室管理システムと連携し、メンバーが在室しているときにノイマンにごはん（ハート）をあげることができます。

---

## 概要

- 研究室のメンバーが在室しているほど、ノイマンが成長します
- メンバーが誰もいない時間が長く続くと、ノイマンが悲しそうになります
- ノイマンは全メンバーで共有する1匹です

---

## 成長システム

### 与えるハート量（feeding rate）

在室人数に応じて、1回あたりのハート量が変化します。
人数が増えるほどハート量は増えます。

```
f(n) = 2 × (1 − 0.5^n)

1人 → 1.00 / tick
2人 → 1.50 / tick
3人 → 1.75 / tick
4人 → 1.875 / tick
...
∞人 → 2.00 / tick（上限）
```

### 成長ステージ

| ステージ | 必要ポイント |
|---------|------------|
| Egg     | 0〜        |
| Sprout  | 300〜      |
| Small   | 1000〜     |
| Growing | 2500〜     |
| Adult   | 5000〜     |

### ノイマンの気分

| 気分    | 条件 |
|--------|------|
| 😊 Happy   | 誰かが在室中 |
| 😐 Neutral | 誰もいないが、最後の在室から時間が経っていない |
| 😢 Sad     | 誰もいない状態が一定時間以上続いている |

```
└── api.js       # 在室 API（現在はモック）
```

---

## 設定（config.js）

| 項目 | 説明 | デフォルト |
|-----|------|-----------|
| `SADNESS_THRESHOLD_HOURS` | 在室者がいない状態が何時間続くとノイマンが悲しくなるか | `4`（時間） |
| `POLL_INTERVAL_MS` | API を取得する間隔 | `10000`（10秒） |
| `MEMBER_COUNT` | 研究室のメンバー数 | `13` |
| `GROWTH_PER_TICK` | 給餌量の倍率 | `1` |
| `MAX_GROWTH_POINTS` | 最大ポイント | `10000` |

## 実際の API への切り替え方

`api.js` の `fetchPresence()` 関数内のコメントを参考に、モック実装を本物の API に置き換えてください。

```js
async function fetchPresence() {
  const res = await fetch('YOUR_API_URL');
  const json = await res.json();
  return json.data;
}
```

API のレスポンス形式：

```json
{
  "data": [
    { "id": "1", "state": "0" },
    { "id": "2", "state": "3" },
    ...
  ]
}
```

`state` が `"0"` または `"1"` のメンバーを在室と判定します。

## マスコット画像・メンバー写真の差し替え方

### ノイマン画像

`index.html` のノイマン部分（`#pet-wrapper` 内）のコメントを参照してください。
現在の CSS 製キャラクターを `<img>` タグに置き換えるだけで対応できます。

### メンバーアバター

`index.html` 内の `M1`〜`M13` のアバター要素に、実際の名前や写真を設定してください。
写真は CSS の `background-image` で指定できます。

---

## 起動方法

サーバー不要です。`index.html` をブラウザで直接開くだけで動作します。

> ただし、実際の在室 API は研究室の内部ネットワーク上でのみ利用可能です。
